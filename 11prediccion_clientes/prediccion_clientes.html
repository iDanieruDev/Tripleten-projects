Descripción del proyecto¶

La compañía de seguros Sure Tomorrow quiere resolver varias tareas con la ayuda de machine learning y nos pide evaluar:

  • Tarea 1: encontrar clientes que sean similares a un cliente determinado. Esto ayudará a los agentes de la compañía con el marketing.
  • Tarea 2: predecir la probabilidad de que un nuevo cliente reciba una prestación del seguro. ¿Puede un modelo de predictivo funcionar mejor que un modelo dummy?
  • Tarea 3: predecir el número de prestaciones de seguro que un nuevo cliente pueda recibir utilizando un modelo de regresión lineal.
  • Tarea 4: proteger los datos personales de los clientes sin afectar al modelo del ejercicio anterior. Es necesario desarrollar un algoritmo de transformación de datos que dificulte la recuperación de la información personal si los datos caen en manos equivocadas. Esto se denomina enmascaramiento u ofuscación de datos. Pero los datos deben protegerse de tal manera que no se vea afectada la calidad de los modelos de machine learning. No es necesario elegir el mejor modelo, basta con demostrar que el algoritmo funciona correctamente.

Tarea 1. Clientes similares¶

En el lenguaje de ML, existe un procedimiento que devuelve los k vecinos más cercanos para un objeto dado basándose en la distancia entre ellos.

Para resolver la tarea, probaremos dos metricas diferentes de distancia, la distancia euclidiana y la distancia por bloques (Manhattan).

A continuación escribiré una función que devuelva los k vecinos más cercanos para un $n^{th}$ objeto basándose en una métrica de distancia especificada. A la hora de realizar esta tarea no consideraremos el número de prestaciones de seguro recibidas. Para realizar esta tarea, usaré un algoritmo kNN ya existente de la libreria scikit-learn.

Una vez creada la función, la probaré para cuatro combinaciones distintas, las cuales son:

  • Distancia Euclidiana con datos normales
  • Distancias Euclidiana con datos escalados (normalizados)
  • Distancia Manhattan con datos normales
  • Distancias Manhattan con datos escalados (normalizados)

Escalar los datos consiste en normalizar todos los datos entre valores de 0 y 1, donde 0 corresponde al valor mínimo del dataset y 1 el valor máximo del dataset. Esto se hace para evitar que una caracteristica tenga mas peso que otra al momento de la evaluación. Para el escalado de datos utilizaré la herramienta MaxAbsScaler de scikit-learn.

In [16]:
feature_names = ['gender', 'age', 'income', 'family_members']
#Creamos nuestra función para deterninar los k vecinos mas cercanos
def get_knn(df, n, k, metric):
    
    """
    Devuelve los k vecinos más cercanos

    :param df: DataFrame de pandas utilizado para encontrar objetos similares dentro del mismo lugar
    :param n: indice del objeto para los que se buscan los "k" vecinos más cercanos
    :param k: número de vecinos más cercanos a devolver
    :param métrica: nombre de la métrica de distancia    """
    
    
    nbrs = NearestNeighbors(metric=metric) 
    nbrs.fit(df[feature_names])    
        
    nbrs_distances, nbrs_indices = nbrs.kneighbors([df.iloc[n][feature_names]], k, return_distance=True)

    df_res = pd.concat([
        df.iloc[nbrs_indices[0]], 
        pd.DataFrame(nbrs_distances.T, index=nbrs_indices[0], columns=['distance'])
        ], axis=1)
    
    return df_res

Tarea 2. ¿Es probable que el cliente reciba una prestación del seguro?¶

En términos de machine learning podemos considerarlo como una tarea de clasificación binaria.

Con el valor de insurance_benefits, evaluaré si el enfoque de clasificación kNN puede funcionar mejor que el modelo dummy.

Pasos:

  • Construiré un clasificador basado en KNN y medire su calidad con la métrica F1 para k=1...10 tanto para los datos originales como para los escalados. Observaremos cómo k puede influir en la métrica de evaluación y si el escalado de los datos provoca alguna diferencia. Para esta tarea utilizaré una implementación ya existente del algoritmo de clasificación kNN de scikit-learn
  • Construiré un modelo dummy que, en este caso, es simplemente un modelo aleatorio. Este modelo devolverá "1" con cuatros probabilidades diferentes:
    • Probabilidad 0 (solo ceros)
    • Probabilidad de pagar cualquier prestación del seguro (ver formula mas abajo)
    • Probabilidad 0.5 (mitad de ceros y mitad de unos)
    • Probabilidad 1 (solo unos)

La probabilidad de pagar cualquier prestación del seguro puede definirse como $$ P\{\text{prestación de seguro recibida}\}=\frac{\text{número de clientes que han recibido alguna prestación de seguro}}{\text{número total de clientes}}. $$

Para las etapas de entrenamiento/prueba, dividiré los datos respetando la proporción 70:30.

Nuestra variable objetivo no se encuentra balanceada ya que tenemos muchas clasificaciones 0 y muy pocas 1. Es necesario balancear para que el desempeño de nuestro modelo sea adecuado. Para realizar el balanceo, aplicaré dos tecnicas, submuestreo y sobremuestreo, la primera elimina un pequeño porcentaje los valores objetivo (ceros) que se encuentra en mayor cantidad, mientras que la segunda, repite algunas muestras de los valores objetivo que se encuentran en menor cantidad (unos). Las funciones que aplican estas tecnicas serán definidas a continuación:

In [25]:
#Submuestreo
def downsample(features, target, fraction):
    features_zeros = features[target == 0]
    features_ones = features[target == 1]
    target_zeros = target[target == 0]
    target_ones = target[target == 1]

    features_downsampled = pd.concat(
        [features_zeros.sample(frac=fraction, random_state=12345)]
        + [features_ones])
    
    target_downsampled = pd.concat(
        [target_zeros.sample(frac=fraction, random_state=12345)]
        + [target_ones])

    features_downsampled, target_downsampled = shuffle(features_downsampled, target_downsampled, random_state=12345)

    return features_downsampled, target_downsampled
In [26]:
#Sobremuestreo
def upsample(features, target, repeat):
    features_zeros = features[target == 0]
    features_ones = features[target == 1]
    target_zeros = target[target == 0]
    target_ones = target[target == 1]

    features_upsampled = pd.concat([features_zeros] + [features_ones] * repeat)
    target_upsampled = pd.concat([target_zeros] + [target_ones] * repeat)

    features_upsampled, target_upsampled = shuffle(
        features_upsampled, target_upsampled, random_state=12345)

    return features_upsampled, target_upsampled

def rnd_model_predict(P, size, seed=42):
    rng = np.random.default_rng(seed=seed)
    return rng.binomial(n=1, p=P, size=size)

Tarea 3. Regresión (con regresión lineal)¶

A continuación, en vez de reducir insurance_benefits a un problema de clasificación simple, usaré un modelo de regresión lineal con insurance_benefits como objetivo. Para evaluar el modelo, lo evalurea con la metrica RECM.

Para esto, construiré una clase personalizada de regresión lineal teniendo en cuenta lo siguiente:

Denotemos

  • $X$: matriz de características; cada fila es un caso, cada columna es una característica, la primera columna está formada por unidades
  • $y$ — objetivo (un vector)
  • $\hat{y}$ — objetivo estimado (un vector)
  • $w$ — vector de pesos

La tarea de regresión lineal en el lenguaje de las matrices puede formularse así: $$ y = Xw $$

El objetivo de entrenamiento es entonces encontrar esa $w$ que minimice la distancia L2 (ECM) entre $Xw$ y $y$:

$$ \min_w d_2(Xw, y) \quad \text{or} \quad \min_w \text{MSE}(Xw, y) $$

Parece que hay una solución analítica para lo anteriormente expuesto: $$ w = (X^T X)^{-1} X^T y $$

La fórmula anterior puede servir para encontrar los pesos $w$ y estos últimos pueden utilizarse para calcular los valores predichos $$ \hat{y} = X_{val}w $$

class MyLinearRegression:
    
    def __init__(self):
        
        self.weights = None
    
    def fit(self, X, y):
        
        # añadir las unidades
        X2 = np.append(np.ones([len(X), 1]), X, axis=1)
        #self.weights = np.dot(X2.T,X2) # X transpuesta por X
        #self.weights = np.linalg.inv(self.weights) #Invierto el resultado anterior
        #self.weights = np.dot(self.weights,X2.T) #Lo multiplico con X transpuesta
        #self.weights = np.dot(self.weights,y) #Lo multiplico por el vector y
        
        self.weights = np.linalg.inv(X2.T.dot(X2)).dot(X2.T).dot(y) #Lo mismo que esta comentado arriba pero en menos pasos
        
        self.w0 = self.weights[0]
        self.weights = self.weights[1:]    
        
    def predict(self, X):
        
        # añadir las unidades
        X2 = X
        y_pred = X2.dot(self.weights) + self.w0
        
        return y_pred

Prueba de que la ofuscación de datos puede funcionar con regresión lineal¶

La tarea de regresión se ha resuelto con la regresión lineal. A continuación demostraré analiticamente que el método de ofuscación no afectará a la regresión lineal en términos de valores predichos, es decir, que sus valores seguirán siendo los mismos tanto para una matriz normal, como para una ofuscada (codificada).

Una vez ofuscados los datos, tenemos $X \times P$ en lugar de tener solo $X$. En consecuencia, hay otros pesos $w_P$ como $$ w = (X^T X)^{-1} X^T y \quad \Rightarrow \quad w_P = [(XP)^T XP]^{-1} (XP)^T y $$

Prueba analítica

Si $w_P = [(XP)^T XP]^{-1} (XP)^T y$

Simplificando:

$ w_P = [(XP)^T XP]^{-1} (XP)^T y \quad/ *[(XP)^T XP] $ (a la izquierda)

$ [(XP)^T XP]w_P = [(XP)^T XP][(XP)^T XP]^{-1} (XP)^T y $

Donde $ I = [(XP)^T XP][(XP)^T XP]^{-1} $

Entonces $ (XP)^T XPw_P = (XP)^T y $

Simplificando (multiplicando por $[(XP)^T]^{-1}$ a la izquierda) y reordenando

$$ \begin{array}{|c|}\hline y =(XP)w_P \\ \hline \end{array} $$

Y como sabemos de antemano que:

$$ \begin{array}{|c|}\hline y = Xw \\ \hline \end{array} $$

Entonces la relacion en $w$ y $w_p$ es:

$$ \begin{array}{|c|}\hline w = Pw_P \\ \hline \end{array} $$

Considerando lo anterior:

Si tenemos $X$ y $X'$ donde $X'=XP$

Sabemos que la predición de la matriz $X$ es:

$ \hat{y} = Xw$ (1)

Mientras que la predicción de la matriz $X'$ (X ofuscada) es:

$ \hat{y_P} = X'w_P $

Entonces si reemplazamos $w = Pw_P$ en (1), obtenemos

$ \hat{y} = Xw = XPw_P = X'w_P = \hat{y_P} $

Por lo tanto, las predicciones del modelo entrenado con la matriz normal y el el modelo entrenado con la matriz ofuscada son equivalentes. $$ \begin{array}{|c|}\hline \hat{y} = \hat{y_P} \\ \hline \end{array} $$

Como vimos en la demostración anterior, la relación entre el vector $w$ y el vector $w_p$ es la multiplicación por la matriz $P$. Por lo tanto, los valores predichos tanto con el vector $w$ como con $w_p$ entregaran como resultado una predicción correcta.

Puedes escribir fórmulas en tu Jupyter Notebook utilizando un lenguaje de marcado proporcionado por un sistema de publicación de alta calidad llamado $\LaTeX$ (se pronuncia como "Lah-tech"). Las fórmulas se verán como las de los libros de texto.

Para incorporar una fórmula a un texto, pon el signo de dólar (\$) antes y después del texto de la fórmula, por ejemplo: $\frac{1}{2} \times \frac{3}{2} = \frac{3}{4}$ or $y = x^2, x \ge 1$.

Si una fórmula debe estar en el mismo párrafo, pon el doble signo de dólar (\$\\$) antes y después del texto de la fórmula, por ejemplo: $$ \bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i. $$

El lenguaje de marcado de LaTeX es muy popular entre las personas que utilizan fórmulas en sus artículos, libros y textos. Puede resultar complicado, pero sus fundamentos son sencillos. Consulta esta ficha de ayuda (materiales en inglés) de dos páginas para aprender a componer las fórmulas más comunes.

Apéndice B: Propiedades de las matrices¶

Las matrices tienen muchas propiedades en cuanto al álgebra lineal. Aquí se enumeran algunas de ellas que pueden ayudarte a la hora de realizar la prueba analítica de este proyecto.

Distributividad$A(B+C)=AB+AC$
No conmutatividad$AB \neq BA$
Propiedad asociativa de la multiplicación$(AB)C = A(BC)$
Propiedad de identidad multiplicativa$IA = AI = A$
$A^{-1}A = AA^{-1} = I$
$(AB)^{-1} = B^{-1}A^{-1}$
Reversibilidad de la transposición de un producto de matrices,$(AB)^T = B^TA^T$
