Descripción del proyecto¶

La compaa de seguros Sure Tomorrow quiere resolver varias tareas con la ayuda de machine learning y nos pide evaluar:

  • Tarea 1: encontrar clientes que sean similares a un cliente determinado. Esto ayudar a los agentes de la compaa con el marketing.
  • Tarea 2: predecir la probabilidad de que un nuevo cliente reciba una prestacin del seguro. Puede un modelo de predictivo funcionar mejor que un modelo dummy?
  • Tarea 3: predecir el nmero de prestaciones de seguro que un nuevo cliente pueda recibir utilizando un modelo de regresin lineal.
  • Tarea 4: proteger los datos personales de los clientes sin afectar al modelo del ejercicio anterior. Es necesario desarrollar un algoritmo de transformacin de datos que dificulte la recuperacin de la informacin personal si los datos caen en manos equivocadas. Esto se denomina enmascaramiento u ofuscacin de datos. Pero los datos deben protegerse de tal manera que no se vea afectada la calidad de los modelos de machine learning. No es necesario elegir el mejor modelo, basta con demostrar que el algoritmo funciona correctamente.

Tarea 1. Clientes similares¶

En el lenguaje de ML, existe un procedimiento que devuelve los k vecinos ms cercanos para un objeto dado basndose en la distancia entre ellos.

Para resolver la tarea, probaremos dos metricas diferentes de distancia, la distancia euclidiana y la distancia por bloques (Manhattan).

A continuacin escribir una funcin que devuelva los k vecinos ms cercanos para un $n^{th}$ objeto basndose en una mtrica de distancia especificada. A la hora de realizar esta tarea no consideraremos el nmero de prestaciones de seguro recibidas. Para realizar esta tarea, usar un algoritmo kNN ya existente de la libreria scikit-learn.

Una vez creada la funcin, la probar para cuatro combinaciones distintas, las cuales son:

  • Distancia Euclidiana con datos normales
  • Distancias Euclidiana con datos escalados (normalizados)
  • Distancia Manhattan con datos normales
  • Distancias Manhattan con datos escalados (normalizados)

Escalar los datos consiste en normalizar todos los datos entre valores de 0 y 1, donde 0 corresponde al valor mnimo del dataset y 1 el valor mximo del dataset. Esto se hace para evitar que una caracteristica tenga mas peso que otra al momento de la evaluacin. Para el escalado de datos utilizar la herramienta MaxAbsScaler de scikit-learn.

feature_names = ['gender', 'age', 'income', 'family_members']
#Creamos nuestra funcin para deterninar los k vecinos mas cercanos
def get_knn(df, n, k, metric):
    
    """
    Devuelve los k vecinos ms cercanos

    :param df: DataFrame de pandas utilizado para encontrar objetos similares dentro del mismo lugar
    :param n: indice del objeto para los que se buscan los "k" vecinos ms cercanos
    :param k: nmero de vecinos ms cercanos a devolver
    :param mtrica: nombre de la mtrica de distancia    """
    
    
    nbrs = NearestNeighbors(metric=metric) 
    nbrs.fit(df[feature_names])    
        
    nbrs_distances, nbrs_indices = nbrs.kneighbors([df.iloc[n][feature_names]], k, return_distance=True)

    df_res = pd.concat([
        df.iloc[nbrs_indices[0]], 
        pd.DataFrame(nbrs_distances.T, index=nbrs_indices[0], columns=['distance'])
        ], axis=1)
    
    return df_res

Tarea 2. ¿Es probable que el cliente reciba una prestación del seguro?¶

En trminos de machine learning podemos considerarlo como una tarea de clasificacin binaria.

Con el valor de insurance_benefits, evaluar si el enfoque de clasificacin kNN puede funcionar mejor que el modelo dummy.

Pasos:

  • Construir un clasificador basado en KNN y medire su calidad con la mtrica F1 para k=1...10 tanto para los datos originales como para los escalados. Observaremos cmo k puede influir en la mtrica de evaluacin y si el escalado de los datos provoca alguna diferencia. Para esta tarea utilizar una implementacin ya existente del algoritmo de clasificacin kNN de scikit-learn
  • Construir un modelo dummy que, en este caso, es simplemente un modelo aleatorio. Este modelo devolver "1" con cuatros probabilidades diferentes:

Nuestra variable objetivo no se encuentra balanceada ya que tenemos muchas clasificaciones 0 y muy pocas 1. Es necesario balancear para que el desempeo de nuestro modelo sea adecuado. Para realizar el balanceo, aplicar dos tecnicas, submuestreo y sobremuestreo, la primera elimina un pequeo porcentaje los valores objetivo (ceros) que se encuentra en mayor cantidad, mientras que la segunda, repite algunas muestras de los valores objetivo que se encuentran en menor cantidad (unos). Las funciones que aplican estas tecnicas sern definidas a continuacin:

#Submuestreo
def downsample(features, target, fraction):
    features_zeros = features[target == 0]
    features_ones = features[target == 1]
    target_zeros = target[target == 0]
    target_ones = target[target == 1]

    features_downsampled = pd.concat(
        [features_zeros.sample(frac=fraction, random_state=12345)]
        + [features_ones])
    
    target_downsampled = pd.concat(
        [target_zeros.sample(frac=fraction, random_state=12345)]
        + [target_ones])

    features_downsampled, target_downsampled = shuffle(features_downsampled, target_downsampled, random_state=12345)

    return features_downsampled, target_downsampled
In [26]:
#Sobremuestreo
def upsample(features, target, repeat):
    features_zeros = features[target == 0]
    features_ones = features[target == 1]
    target_zeros = target[target == 0]
    target_ones = target[target == 1]

    features_upsampled = pd.concat([features_zeros] + [features_ones] * repeat)
    target_upsampled = pd.concat([target_zeros] + [target_ones] * repeat)

    features_upsampled, target_upsampled = shuffle(
        features_upsampled, target_upsampled, random_state=12345)

    return features_upsampled, target_upsampled

def rnd_model_predict(P, size, seed=42):
    rng = np.random.default_rng(seed=seed)
    return rng.binomial(n=1, p=P, size=size)

Para esto, construir una clase personalizada de regresin lineal teniendo en cuenta lo siguiente:

Denotemos

  • $X$: matriz de caractersticas; cada fila es un caso, cada columna es una caracterstica, la primera columna est formada por unidades
  • $y$ objetivo (un vector)
  • $\hat{y}$ objetivo estimado (un vector)
  • $w$ vector de pesos

La tarea de regresin lineal en el lenguaje de las matrices puede formularse as: $$ y = Xw $$

El objetivo de entrenamiento es entonces encontrar esa $w$ que minimice la distancia L2 (ECM) entre $Xw$ y $y$:

$$ \min_w d_2(Xw, y) \quad \text{or} \quad \min_w \text{MSE}(Xw, y) $$

Parece que hay una solucin analtica para lo anteriormente expuesto: $$ w = (X^T X)^{-1} X^T y $$

La frmula anterior puede servir para encontrar los pesos $w$ y estos ltimos pueden utilizarse para calcular los valores predichos $$ \hat{y} = X_{val}w $$

class MyLinearRegression:
    
    def __init__(self):
        
        self.weights = None
    
    def fit(self, X, y):
        
        # aadir las unidades
        X2 = np.append(np.ones([len(X), 1]), X, axis=1)
        #self.weights = np.dot(X2.T,X2) # X transpuesta por X
        #self.weights = np.linalg.inv(self.weights) #Invierto el resultado anterior
        #self.weights = np.dot(self.weights,X2.T) #Lo multiplico con X transpuesta
        #self.weights = np.dot(self.weights,y) #Lo multiplico por el vector y
        
        self.weights = np.linalg.inv(X2.T.dot(X2)).dot(X2.T).dot(y) #Lo mismo que esta comentado arriba pero en menos pasos
        
        self.w0 = self.weights[0]
        self.weights = self.weights[1:]    
        
    def predict(self, X):
        
        # aadir las unidades
        X2 = X
        y_pred = X2.dot(self.weights) + self.w0
        
        return y_pred

Prueba de que la ofuscación de datos puede funcionar con regresión lineal¶

La tarea de regresin se ha resuelto con la regresin lineal. A continuacin demostrar analiticamente que el mtodo de ofuscacin no afectar a la regresin lineal en trminos de valores predichos, es decir, que sus valores seguirn siendo los mismos tanto para una matriz normal, como para una ofuscada (codificada).

Una vez ofuscados los datos, tenemos $X \times P$ en lugar de tener solo $X$. En consecuencia, hay otros pesos $w_P$ como $$ w = (X^T X)^{-1} X^T y \quad \Rightarrow \quad w_P = [(XP)^T XP]^{-1} (XP)^T y $$

Prueba analtica

Si $w_P = [(XP)^T XP]^{-1} (XP)^T y$

Simplificando:

$ w_P = [(XP)^T XP]^{-1} (XP)^T y \quad/ *[(XP)^T XP] $ (a la izquierda)

$ [(XP)^T XP]w_P = [(XP)^T XP][(XP)^T XP]^{-1} (XP)^T y $

Donde $ I = [(XP)^T XP][(XP)^T XP]^{-1} $

Entonces $ (XP)^T XPw_P = (XP)^T y $

Simplificando (multiplicando por $[(XP)^T]^{-1}$ a la izquierda) y reordenando

$$ \begin{array}{|c|}\hline y =(XP)w_P \\ \hline \end{array} $$

Y como sabemos de antemano que:

$$ \begin{array}{|c|}\hline y = Xw \\ \hline \end{array} $$

Entonces la relacion en $w$ y $w_p$ es:

$$ \begin{array}{|c|}\hline w = Pw_P \\ \hline \end{array} $$

Considerando lo anterior:

Si tenemos $X$ y $X'$ donde $X'=XP$

Sabemos que la predicin de la matriz $X$ es:

$ \hat{y} = Xw$ (1)

Mientras que la prediccin de la matriz $X'$ (X ofuscada) es:

$ \hat{y_P} = X'w_P $

Entonces si reemplazamos $w = Pw_P$ en (1), obtenemos

$ \hat{y} = Xw = XPw_P = X'w_P = \hat{y_P} $

Por lo tanto, las predicciones del modelo entrenado con la matriz normal y el el modelo entrenado con la matriz ofuscada son equivalentes. $$ \begin{array}{|c|}\hline \hat{y} = \hat{y_P} \\ \hline \end{array} $$

Como vimos en la demostracin anterior, la relacin entre el vector $w$ y el vector $w_p$ es la multiplicacin por la matriz $P$. Por lo tanto, los valores predichos tanto con el vector $w$ como con $w_p$ entregaran como resultado una prediccin correcta.

Si una frmula debe estar en el mismo prrafo, pon el doble signo de dlar (\$\\$) antes y despus del texto de la frmula, por ejemplo: $$ \bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i. $$

El lenguaje de marcado de LaTeX es muy popular entre las personas que utilizan frmulas en sus artculos, libros y textos. Puede resultar complicado, pero sus fundamentos son sencillos. Consulta esta ficha de ayuda (materiales en ingls) de dos pginas para aprender a componer las frmulas ms comunes.
